v1.32 — July 15, 2026 (minor)
  - Practical Tips (Assessment Summary Table) now name the specific
    assessment(s) each tip concerns (e.g., "Assignment 2: ...",
    "Term Project, Final Exam: ..."), so there is no ambiguity about
    which assessment the advice applies to.
  - Assignments: rubric made mandatory — the tip states that a rubric
    is a must (attached to the brief) and that submissions should be
    run through a similarity checker (e.g., Turnitin or the
    institution's approved platform), with the policy stated up front.
  - New "Rubric prepared" checkbox on Assignment / Project /
    Presentation / Viva assessment cards; until ticked, the summary
    table shows an amber "rubric pending — <name>" badge. Rubrics
    added to the CQI tab's Annexure H evidence checklist.
  - New tips: authorship check in the AI era (pair take-home
    assignments/projects with a short viva or in-class task);
    individual accountability in group projects (peer assessment /
    contribution-weighted marks); formative use of quizzes with rapid
    feedback before the next major assessment (Nicol &
    Macfarlane-Dick 2006).
  - Manual and README regenerated with v1.32 stamps.

v1.31 — July 15, 2026 (minor)
  - Assessment Summary Table (Assessments tab): the "Comments from CA
    Perspective" column is replaced by "Practical Tips" — short,
    rule-generated advice on the assessments measuring each CO, from an
    engineering-education perspective: analytic rubrics shared in
    advance (projects/assignments, labs, presentations/vivas),
    authentic exam problems with realistic data and pre-moderated
    marking schemes, parallel optional questions, observation
    checklists for affective/psychomotor COs, and formative use of
    quizzes. Grounded in the assessment literature (Biggs & Tang;
    Wiggins 1990; Gulikers et al. 2004; Jonsson & Svingby 2007;
    Panadero & Jonsson 2013) — see the Manual's new "Authentic
    assessment & rubrics" note. Unassessed COs remain flagged in red;
    alignment analysis stays in the CA tab (no duplication).
  - Manual and README regenerated with v1.31 stamps.

v1.20 — July 14, 2026 (minor)
  - Choice scheme per assessment: declare "students must answer k of the
    optional questions" in the Assessment Plan (question-unit = a group
    or an ungrouped optional item). Rule R5: a student attempting fewer
    than k units has the shortfall counted as zeros, drawn from the
    unattempted units with the highest maximum marks (deterministic,
    conservative). Attempting more than k counts all attempted units.
  - Absent rule: with a declared scheme, a student with no marks
    anywhere in that assessment is treated as absent from it and
    excluded from its calculations (instead of receiving phantom zeros).
  - Full methodology with worked examples published in
    Attainment_Calculation_Methodology.docx (Annex A now implemented).

v1.19 — July 14, 2026 (minor)
  - Assessment Plan: optional items can share a question Group (e.g.,
    sub-parts Q3a/Q3b grouped as "Q3"). If a student attempts any item
    of the group, the whole group counts for that student (unanswered
    sub-parts = 0) — partial answers to a deliberately chosen question
    are penalized. If no item of the group is attempted, the whole group
    is excluded (no penalty). Groups are scoped per assessment; an
    entered 0 counts as an attempt and engages the group.
  - Group shown in the assessment blueprint; low-attainment item
    averages are group-aware.

v1.18 — July 14, 2026 (minor)
  - Assessment Plan: every item now has a "Mandatory" checkbox (default:
    ticked). Fair handling of choice-based exams (e.g., "Q1 mandatory,
    answer any four of Q2-Q7"): a blank mark on a mandatory item counts
    as 0, while a blank on an optional item is excluded from that
    student's CO calculation — no penalty for legitimately skipped
    questions. Students with no assessed marks under a CO are excluded
    from that CO's cohort (per-CO n shown in the Attainment tab), and
    per-student PO scores use only the COs the student was assessed on.
  - Optional items are marked with † in the marks grid and the vetting
    form, "(optional)" in the assessment blueprint, and an "optional"
    badge in the low-attainment diagnostics.
  - Convention: blank = not attempted; an entered 0 = attempted and
    scored zero.

v1 — July 13, 2026 (first release)
  Core:
  - Course management (multiple courses), Course Info, attainment settings.
  - COs with Bloom's taxonomy (C/A/P) and click-to-set CO-PO mapping matrix
    (12 Washington Accord Program Outcomes, strengths 1-3).
  - T&L Activities tab: CQI-aware recommender with a pool of 31 pedagogies
    (extensible with custom activities), per-CO recommendations filtered by
    mode and class size, one-click adoption into delivery methods.
  - Assessment plan: assessments with CO-tagged items and maximum marks.
  - Students & Marks: individual/bulk enrolment (Excel paste), marks grid,
    Empty Template (CSV) / Import Marks (CSV) / Export Marks (CSV).
  Attainment:
  - Direct measurement following the AUST "Direct CO-PO Calculations"
    workbook method (Target Pass Marks 40, KPI 50, per-student PO scores
    weighted by CO allocated marks), with charts.
  - Indirect measurement via CO Exit Survey (tally grid, Google Forms CSV
    import, printable survey form), Likert threshold configurable.
  - Combined attainment with user-editable Direct/Indirect weights
    (default 80/20).
  CQI (BAETE Criteria 3.8 & 5.4.3):
  - Auto-compiled per-CO table of T&L methods, assessment tools, CP/CEA;
    narrative fields (complex problems, effectiveness, tool design,
    question moderation).
  - Six-step CQI-loop status board; action tables two-way linked with
    Course Report 6.1 and Course Specifications 6.1.
  - Low Attainment Analysis linked to the Attainment tab: threshold filter
    for low-scoring students per CO, item-wise average diagnostics,
    analyses w.r.t. (a) T&L activities and (b) assessment techniques,
    and comments.
  - Stakeholder Feedback Register and Annexure H evidence checklist.
  AUST Documents (auto-filled, printable):
  - Course Outline, Course Specifications, Final Examination Vetting Form,
    Direct CO-PO Calculations, CO Exit Survey, Course Report, CQI Report.
  Data:
  - Work on a real .json file with auto-save (File System Access API,
    Chrome/Edge), reconnect flow; localStorage cache; JSON export/import;
    "Duplicate for New Semester (CQI)" carrying action plans forward.
  Documentation:
  - Faculty User Manual (docx) with Appendix A: T&L activity pool
    descriptions and selection justifications.
