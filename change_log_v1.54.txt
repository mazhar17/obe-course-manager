v1.54 — July 15, 2026 (minor)
  - "Copy AI Prompt (CQI narratives & action plans)" now asks for
    exactly what its name promises. The task list previously
    requested low-attainment analyses (which belong to the Low
    Attainment Analysis section) and generic actions; it now targets,
    labelled verbatim for field-by-field paste-back:
    1) "How the teaching-learning methods and assessment tools
       address complex engineering problems and the POs"
    2) "Why these methods and tools are effective and appropriate"
       (citing attainment, survey findings, TLA effectiveness;
       honest about the not-attained COs)
    3) "Process of designing and selecting assessment tools"
    4) "Question moderation / vetting process"
    5) "Improvement actions from this offering (Course Report 6.1)"
       — one block per loop row (CO/PO/PEO, direct/indirect), with
       the strict "No specific action this offering." fallback.
    New evidence in the prompt: CP/CEA tags on the COs, per-assessment
    rubric status, recorded TLA effectiveness, and vetter names from
    the Vetting Form.
  - The printable CQI Report is now explicitly fully auto-generated:
    a screen-only notice states that every section is drawn from the
    CQI tab, Course Specifications and Course Report — nothing to
    fill in on the document; complete the sources and print. The
    narrative boxes preserve line breaks and their empty-state text
    points to the CQI tab.
  - Manual and README regenerated with v1.54 stamps.

v1.46 — July 15, 2026 (minor)
  - Synthesis prompt (Specifications 6.1, "Actions reflected")
    rewritten for depth: the previous "2-4 sentences per loop row"
    cap flattened rich working notes into inadequate summaries. The
    task now demands the same granularity as the notes — one block
    PER CO (direct CO row) and one block PER mapped PO (direct PO
    row), each naming the concrete actions (activities by name,
    assessment items with type and Bloom level, rubric and
    similarity-check actions, marks-weight changes) and the
    recommendation each closes. New constraint: "omit nothing that
    appears in the notes; do not pad rows that have no matching
    action". Indirect rows keep the strict "No specific action this
    offering." fallback. The auto-growing 6.1 boxes accommodate the
    longer text.
  - Manual and README regenerated with v1.46 stamps.

v1.45 — July 15, 2026 (minor)
  - All four AI prompts now require justifications, so faculty can
    evaluate the AI's advice instead of taking it on trust:
    * T&L suggestions prompt — one-sentence justification per
      activity (fit to the CO's Bloom domain/level, which
      recommendation it addresses, feasibility for the class size);
    * Assessment suggestions prompt — one-sentence justification per
      technique (validity / reliability / authenticity gains at that
      CO's level, recommendation addressed);
    * Synthesis prompt ("Actions reflected") — each closure statement
      must explicitly name the recommendation it closes;
    * CQI narratives prompt — every recommended action must carry a
      one-sentence justification tied to the specific evidence (CO/PO
      figures, weak items, survey gaps); no unsupported assertions.
    A shared rule keeps justifications to one sentence — reusable in
    the CQI narratives.
  - CQI tab: the generic "Copy AI Prompt" button is renamed
    "Copy AI Prompt (CQI narratives & action plans)" to make its role
    obvious next to the other, specifically-named prompt buttons.
  - Manual and README regenerated with v1.45 stamps.

v1.20 — July 14, 2026 (minor)
  - Choice scheme per assessment: declare "students must answer k of the
    optional questions" in the Assessment Plan (question-unit = a group
    or an ungrouped optional item). Rule R5: a student attempting fewer
    than k units has the shortfall counted as zeros, drawn from the
    unattempted units with the highest maximum marks (deterministic,
    conservative). Attempting more than k counts all attempted units.
  - Absent rule: with a declared scheme, a student with no marks
    anywhere in that assessment is treated as absent from it and
    excluded from its calculations (instead of receiving phantom zeros).
  - Full methodology with worked examples published in

v1.19 — July 14, 2026 (minor)
